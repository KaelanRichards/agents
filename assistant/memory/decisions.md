## Agent memory lives in the control plane, not Claude auto-memory

**Current truth.** Durable memory is plain markdown under `~/.config/agents/assistant/memory/`,
jj-versioned and shared across tools. Claude Code's built-in auto-memory is disabled
(`autoMemoryEnabled: false`). Hermes auto-loads these files via `hermes-sync` `context_files`;
Claude + Codex get a compact index in `AGENTS.md` (regenerated by `agents-sync`) and open files on
demand.

**Details.** Researched May 2026: git-versioned markdown is the recommended canonical store
(auditable, human + agent read the same bytes, rollbackable). Managed memory vendors (Mem0/Zep/Letta)
and knowledge-graph MCP servers are the wrong tradeoff for a single-user, auditability-first,
corp-managed setup. A retrieval index (SQLite FTS5) is deferred until curated memory is large —
Hermes already has FTS5 search over its own session transcripts.

**Open questions.** Episodic→semantic consolidation (promote session learnings into curated memory)
is unsolved here and across the field — revisit as a scheduled routine if wanted.
